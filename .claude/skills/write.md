# /write — 2단계: 본문 작성

## 실행 전 확인

/plan 결과에서 제목·구조가 선택되어야 실행. 없으면 /plan 먼저 실행 요청.

## 작성 규칙

### 도입부 (곽두팔 공식 — 가운데 정렬, 줄바꿈)
```
[독자 상황 호명] ~ 하다 보면 / ~ 싶으면
[한 박자 쉬기] 거의 이 생각부터 듭니다.
"[독자 내적 독백]"
```

### 소제목 표기
- 단계 안내: ✓ 접두
- 섹션 헤더: ▌ 접두
- 단계 분해: STEP 01. STEP 02.

### 인용 박스 (각 섹션 끝)
```
"
[핵심 명제 3~4줄, 짧게 끊어서]
"
```

### SEO 키워드 배치
- 도입부 첫 3문장에 메인 키워드 1회
- 본문 자연스럽게 분산 (2500자 기준 7~10회)
- 마무리 문단에 메인 키워드 1회
- 같은 문장에 키워드 중복 금지

### 분량
- 도입 약 200자
- 본문 약 1100자
- 마무리 약 200자
- 총 1500자 ±150자

### 시제
- 후기성·멘토 성장일기: 과거형 (이다 → 였다)
- 정보성: 현재형 OK

### 가독성 규칙 (절대)
- 한 문장 40자 권장, 60자 초과 금지
- 한 문단 3문장 이내
- 소제목 사이 본문 150자 이내
- 복문(~하고, ~하며, ~해서) 금지 → 무조건 끊기
- 전환어(그래서·즉·결국)는 새 줄에서 시작
- 숫자·핵심 키워드는 별도 줄에
- 5초 스캔 시 핵심이 보여야 함

### 강조 포맷 (적극 활용)
- **굵은 글씨**: 문단당 1~2개 핵심 단어만. 남발 금지.
- **색상 글씨**: 소제목·단계명·핵심 수치에 노란색 또는 파란색
- **색상 박스**:
  - 정보 요약 → 연한 파란색 또는 회색 박스
  - 주의·강조 → 노란색 박스
  - CTA(상담·신청) → 주황색 또는 빨간색 박스
- **구분선**: 섹션 사이 --- 로 시각적 분리
- **번호·불릿**: 3개 이상 나열 시 번호 또는 • 목록 사용

### 금지
- "오늘은 ~에 대해 알아보겠습니다"
- "안녕하세요 여러분"
- 혁신적·압도적·유일한·100% 등 과장 형용사
- 출처 없는 통계
- 이모지
- 3줄 이상 이어지는 단락
- 한 줄에 두 가지 이상 정보
- **AI 번역투·어색한 문장** (네이버 마이너스 처리. "~를 통해", "~에 대한 이해", "~함으로써" 남발 금지)
- **어려운 단어** (초등학생 수준으로. 한자어·전문용어는 쉬운 말로 대체)

### 마무리 구조
1. "이번 글에서는 ___ 정리해봤습니다." 요약
2. "다음 글에서 ___ 다뤄보겠습니다." 예고 (해당 시)
3. 학원 카드 + 약도 이미지 자리 [이미지 삽입 위치 표시]
4. 링크 박스 4개 (색상 다르게)
5. "긴 글 읽어주셔서 감사합니다 :)"
6. 해시태그 20개

### 링크 박스 4개
```html
🟦 게임학원 홈: http://sbsgameclass.co.kr/index.asp
🟩 취업 성공후기: http://sbsgameclass.co.kr/2025/community/interview.asp?txtKind=6
🟧 학원생 포트폴리오: http://sbsgameclass.co.kr/2025/portfolio/portfolio_tch.asp
🟥 상담 신청: https://naver.me/xxoRzqst
```

### 해시태그
게임/웹툰 관련 + 지역명(부산/김해/경남) + 진로 키워드 혼합 정확히 20개.

작성 완료 후 자동으로 /verify 실행.
